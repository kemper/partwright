## Human

Observed that most AI API functions (especially paint) are themselves
operations that could live as code in the editor, durably with the model —
rather than as ephemeral tool calls captured only in mesh. Asked to
investigate, then (after a design discussion and a `.plans/` design doc)

## Assistant

## Key decisions

**Framing.** Paint is already an operation log (descriptors re-resolved every
run), stored in a hidden sidecar (`version.colorRegions`). The feature isn't
"mesh → operations"; it's "relocate the operation log into the code." The
proven template is `api.label(shape, name, { color })` — color declared in
code, resolved post-run, rendered as the model-color underlay, never
serialized.

**Mechanism (reuse, don't invent).** Added an `api.paint.*` namespace to the
manifold-js sandbox (`box` / `slab` / `cylinder` / `label`). Calls **record** a
`RegionDescriptor` + color during evaluation and mutate nothing. The recorded
ops cross the worker boundary in `MeshResult.paintOps` (plain serializable
objects, alongside the existing `labelColors`). On the main thread they're
resolved through the *existing* `resolveDescriptorTriangles` and fed into the
model-color underlay via a generalized `setModelColorRegions` (now carries each
decl's real descriptor, defaulting to `byLabel`). Zero new persistence, bounded
perf (same resolver path as a user paint region), and `getRegions()` stays
empty — the code is the source of truth.

**Layering.** Typed `paintOps.descriptor` as `unknown` in `geometry/types.ts`
to avoid a `regions.ts ↔ geometry/types.ts` module cycle; the worker engine
uses a type-only `RegionDescriptor` import (no cycle — `regions` doesn't import
the engine), and the main thread casts back. `npm run lint:deps` confirms the
graph stays acyclic.

**Edge case fixed (latent).** The smooth-brush refine paths subdivide the
working mesh but only re-resolved *user* regions, leaving the code/label
underlay stale. Added `reresolveModelRegions` + `setModelRegionTriangles` and
called them in all four mesh-mutating paint paths (sync/async rebuild +
sync/async stroke append). This also fixes the pre-existing
`api.label({color})` + smooth-brush combination.

**Scope.** Implemented the code→render direction (the core "code as artifact"
value). Tier-B coordinate/image paint and UI→code write-back are deliberately
out of scope (see `.plans/code-as-artifact.md`). manifold-js only for now;
documented the boundary in `ai.md` / `ai/colors.md`.

**Verification.** Browser screenshot confirmed slab+box+cylinder render with
later-op-wins overlap on a refined cube; added `tests/paint-in-code.spec.ts`
(golden path + argument-validation cases).

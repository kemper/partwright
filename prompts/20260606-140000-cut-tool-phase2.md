## Key decisions

**Root cause of all crashes: `insertBefore` with non-direct-child reference nodes.**

The project recently merged a toolbar-grouping feature from `origin/main` that wraps
buttons inside `<div>` group containers. Three call sites used the pattern
`parent.insertBefore(newBtn, existingBtn)` where `existingBtn` was found via
`querySelector` — which can return a node nested anywhere in the subtree, not
necessarily a direct child of `parent`. `insertBefore` requires the reference node
to be a direct child of the parent, so all three sites were throwing
`NotFoundError: The node before which the new node is to be inserted is not a child
of this node`.

**Fix pattern**: Replace `parent.insertBefore(newBtn, anchor)` with
`anchor?.parentNode ? anchor.parentNode.insertBefore(newBtn, anchor) : parent.appendChild(newBtn)`.
This uses the anchor's actual parent regardless of how deeply nested it is.

**Three affected sites** (all found by successive Playwright crash reproduction):
1. `src/main.ts` — `customizeBtn` insertion before `#viewport-view-group`
2. `src/main.ts` — `reliefViewportBtn` insertion before a relief anchor
3. `src/ui/cutUI.ts` — `cutBtn` insertion before `#simplify-toggle`

Sites 1 and 2 were fixed in earlier commits; site 3 was discovered when CI still
showed 129 failures despite those fixes. Reproduced locally by running the worktree
dev server and capturing `pageerror` / `console` events with a Playwright scratch spec.

**Exploded preview implementation**:

- Extract cut normal from `mat4x3[6..8]` (column 2 = local Z axis = cut plane normal).
- Offset = 8% of the max bounding box dimension of the base mesh — large enough to
  reveal the cut face, small enough not to look absurd.
- `buildExplodedMesh()` in `src/cut/cutWorker.ts` translates kept/complement meshes
  along ±normal and merges into a single display mesh.

**Part Mode Toggle**:
- `CutResultMode = 'separate' | 'combined'` — controls how `save()` assembles the
  result. Separate = one part per Manifold component. Combined = `mergeMeshData()`
  concatenates all components.
- Default is `'separate'` to match the expected use case (splitting one solid into
  two halves for assembly).

**`notifyChange()` on `activate()` distortion**:
Removed the `notifyChange()` call from `cutGizmo.ts`'s `activate()`. The distortion
was the viewport updating in response to the gizmo appearing before any cut had
actually been applied.

**Merge conflict resolution**:
A curly-quote string delimiter (U+2018/U+2019) was introduced during the
`origin/main` merge at a `querySelector('#viewport-view-group')` call site.
Fixed with a targeted Python byte-position replacement rather than a regex,
since the curly quotes broke every string parser that tried to read the line.

## Key decisions

**Closed the gap by adding the feature, not just re-wording the docs.**
The retro had patched the docs to call out a manifold-js-only caveat; the
user asked to instead bring BREP to parity. `BREP.label(shape, name)` now
accepts the same optional `{ color }` 3rd arg as manifold-js `api.label`
(hex string or `[r,g,b]` 0..1), parsed with the shared
`parseLabelColor` helper so validation/error messages match exactly.

**Color is keyed by label name, so propagation is trivial and exact.**
Unlike `_faceLabels` (face-hashcode keyed) and `_labelSignatures`
(geometry keyed), a `_labelColors: Map<name, [r,g,b]>` carries forward
unchanged through every op. Threaded a 4th `labelColors` param through
`wrap`/`wrapInner` and every op's `wrap(...)` call (fillet, chamfer,
translate, rotate, shell, linearPattern, reduceShapes single-clone);
booleans merge both sides' maps (self wins on name collision, matching the
left-to-right reduce model).

**Reused the existing engine-agnostic color underlay plumbing.**
`MeshResult.labelColors` already existed (manifold-js emitted it) and the
worker already (de)serializes it, and the main-thread run handler already
resolves it against `labelMap` into the model-color underlay regardless of
engine. So the wiring was just: `extractLabelColors(shape)` →
`replicad.ts` attaches `labelColors` to both return branches. For the
Phase C path (`api.BREP.*` inside a manifold-js session), added a
`pendingToManifoldLabelColors` side-channel mirroring the existing
`pendingToManifoldLabels`, drained by the manifold-js engine and merged
into its own `labelColors` (a same-name `api.label` color wins).

**Docs updated to match reality.** `replicad.md` BREP.label block now
shows the `{ color }` examples; `colors.md` notes BREP has the same arg
(with the honest caveat that triangle *coverage* across fused composites
is best-effort per the spatial-signature gotcha, while the color itself is
exact) and clarifies SCAD `label()` still has no color option.

**Verification:** added three e2e tests (Phase A replicad self-coloring via
`getModelColors()`, Phase C `toManifold` flow, bad-color rejection),
screenshotted a colored BREP model in the viewport, and confirmed
`lint:deps` stays acyclic despite the new `parseLabelColor` import into
`brepRuntime.ts`.

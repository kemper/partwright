## Key decisions

**Updated three existing figures in place; added two new face busts.** The
gaze feature is most legible on a face, so the new entries are head-and-shoulders
**busts** (low-tri, face-filling): `bust_cross_eyed` (per-eye `gazeL:'right' +
gazeR:'left'` → cross-eyed, the headline per-eye showcase) and `bust_stargazer`
(`gaze:'up'` + chin-up head pose → dreamy upward look). The three updated
figures got a characterful gaze layered onto their existing pose: the reader
looks `'down'` at the page, the princess glances `'left'`, the waving kid looks
`'up'` at whoever they're greeting.

**Busts use `api.paint.*` so colour bakes into the geometry — no palette file.**
Unlike the `.label()`+palette figures (which I re-baked with their committed
`public/catalog/palettes/*.json`), the new busts paint in code, so the thumbnail
bakes coloured directly. Pinned their thumbnail camera front-on
(`THUMB_AZIMUTH=0`) so the gaze direction reads in the grid; the existing
figures kept their default 3/4 camera.

**Thumbnails are build artifacts, not committed.** The catalog UI serves
`/catalog/thumbs/<id>.<hash>.png`, generated by the vite prerender plugin from
each entry JSON's embedded base64 `version.thumbnail` at server start/build —
and `public/catalog/thumbs/` is gitignored. So the commit carries only the
example sources, the baked `.partwright.json` (which holds the base64
thumbnail), and the `manifest.json` registration; production regenerates the
hashed thumbs. Verified in-browser after a dev-server restart that both new
tiles render their thumbnails and the manifest count rose 131 → 133
(`lint:catalog` green).

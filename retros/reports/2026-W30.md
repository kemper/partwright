## Top friction (ranked by cross-session frequency × cost)

1. **[tooling] Inverse-CAD multi-agent orchestration has no driver/bookkeeping tool** — 3 sessions (`20260703-inverse-cad-v2`, `20260704-inverse-cad-armor-genericity`, `20260705-benchy-foreign-mesh`). Symptoms: a part silently never got an agent assigned (caught only by a final manual sweep), no way to tell "agent finished" from "best/ exists" (led to baking an interim, still-polishing result), and three agents died mid-part to rate limits with resume only cheap because of ad hoc state files. All three sessions independently describe wanting the same status/driver script. **Action: backlog item below** (needs a new script, not a doc change).
2. **[tooling] No warm-browser loop for iterative UI development** — 2 sessions (`2026-07-06-multi-part-assembly-view`, `20260707-3mf-packing-strategy`), the second explicitly noting "same wish as prior retros." Each verification round pays a cold ~4s WASM boot + fresh session build inside Playwright; one session iterated ~5 times and estimated minutes of dead wall-clock lost. The repo already has this exact pattern for model rendering (`bin/partwright.mjs daemon start`) — the ask is a general-purpose UI equivalent. **Action: backlog item below.**
3. **[testing] Rail/list-drag Playwright tests silently miss at the default viewport** — 1 session (`20260706-part-grouping`), but concrete and costly (two burned runs plus a debug scratch before finding the rows were scrolled out of the clip at 1280×720). **Action: applied** — added as gotcha #6 in `docs/playwright-guide.md`.
4. **[figures/tooling] Component-count tools disagree on degenerate meshes** — 1 session (`20260705-multipart-kits-overview`): `decompose()`, the app's `statsComputation` count, and a welded-mesh census gave different answers (and `decompose()` itself flip-flopped run to run) on debris-adjacent meshes, costing ~8 bake round-trips (~2 min each) to sort out. **Action: backlog item below**, extending the existing `componentCount`-trust guidance in CLAUDE.md rather than replacing it.
5. **[process] Stop hook fires repeatedly during long unattended multi-agent waits with nothing to reconcile** — 1 session (`20260704-inverse-cad-armor-genericity`), ~20 firings. Real friction, but a hook/`settings.json` change is out of scope for a direct doc edit. **Action: backlog item below.**

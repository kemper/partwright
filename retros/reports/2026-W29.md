## Top friction (ranked)

1. **[tooling] No warm-browser loop for UI iteration (only model-render has one)** — 2 sessions (assembly-view, 3mf-packing-strategy), independently calling it "the biggest wall-clock lever." Every UI iteration round pays a cold ~4s WASM boot + session build inside a fresh Playwright process; `model:preview`/the `partwright daemon` already solved this exact problem for model-code rendering but nothing analogous exists for driving `window.partwright` UI flows + screenshotting. → **Backlog** (needs new tooling, see below).

2. **[inverse-cad] Orchestration/state-tracking gaps across multi-agent convergence runs** — 3 sessions (v2, armor-genericity, benchy). Concretely: no status driver so an unassigned part goes unnoticed until a final manual sweep; no "agent finished" vs "best exists" signal so an orchestrator or a bake can consume a moving/interim best; a mid-run `gates.mjs` fix silently stales an already-"passing" stored best with no version stamp to catch it. → **Backlog** (needs new tooling, see below).

3. **[testing] Two documented-but-insufficient gotchas in the existing docs, both directly costing a wasted round this week** — 1 vote each, but both are corrections to prose already in CLAUDE.md that turned out to be incomplete, so applying the fix is low-risk/low-cost rather than "hardening a one-off":
   - The NUL-byte `main.ts` section told agents to use `grep -a`/`rg -a` but didn't say the *dedicated `Grep` tool* has no such flag — 3mf-packing-strategy burned a wasted attempt discovering that the only escape is `Bash` + `grep -an`. **Applied**: CLAUDE.md now says this explicitly.
   - `showToast`'s call signature was described only by example (`{ log: false }`, `{ source }`) without ever stating it's always `(message, optionsObject)` — mesh-to-code-in-app guessed a positional-variant form and paid a typecheck round-trip. **Applied**: CLAUDE.md now states the signature directly.

4. **[packing/export] No visual QC path for plate/pack layout** — 2 sessions (3mf-packing-strategy #910, bambu-plate-layout-packing #904), both independently wanting "the packing equivalent of `model:preview`" — currently layout bugs are verified by parsing `<item>` transforms out of 3MF XML and asserting bounds numerically, with no way to *look* at the result the way geometry has `model:preview`. → **Backlog**.

5. **[testing] Rail/list drag e2e specs silently miss at the default viewport** — 1 vote (part-grouping), but the exact same failure shape as an already-documented gotcha (`docs/playwright-guide.md` #2, default viewport clipping the AI panel). Same file, same pattern, cheap fix. **Applied**: added item #6 to `docs/playwright-guide.md`.

## Backlog (needs a human or the tooling/linter agent)

- [ ] **General warm-browser UI iteration harness.** Extend the existing `bin/partwright.mjs daemon` (currently scoped to model-code rendering via `partwright render`/`iterate`) — or add a sibling command — to drive arbitrary `window.partwright` calls + screenshot against an already-warm browser+WASM session, so UI feature iteration doesn't pay a full cold Playwright boot per look. Raised by 2 sessions (assembly-view, 3mf-packing-strategy); this is the single most-repeated "longed for" this cycle.
- [ ] **Inverse-CAD convergence status driver** (`scripts/inverse-cad/status.mjs`, never built per the original design). Needs: a live table of per-part state (assigned / in-progress / converged / stale), auto-refill of idle agent slots, and completion detection — replacing the current manual pool management across ~20 agent launches. Raised by armor-genericity; related bookkeeping gaps also raised by v2 and benchy.
- [ ] **"Agent finished" vs "best exists" signal in `turn.mjs`.** An orchestrator or a catalog bake currently can't distinguish a part that's done from one whose `best/` is still being polished mid-run — benchy-foreign-mesh baked an interim best because of this ambiguity. Needs a `state.json`/lockfile convention `turn.mjs` writes and orchestrators check before consuming `best/`.
- [ ] **Stamp `gates.mjs`'s version/hash into `best/metrics.json`.** When gate logic changes (v2 hit this with a genus-convention fix), already-"passing" stored bests should flag as stale for re-validation instead of silently staying marked green. Raised by inverse-cad-v2.
- [ ] **Headless plate/pack-layout rasterizer** — render a build-plate arrangement (part footprints on the bed) to a PNG, the packing analogue of `model:preview`. Raised independently by 3mf-packing-strategy and bambu-plate-layout-packing; both currently verify packing only by parsing 3MF XML transforms and asserting numeric bounds.
- [ ] **Multi-method geometry-count disagreement explainer.** `Manifold.decompose()`, the app's `statsComputation` count, and a welded-mesh census disagree on degenerate meshes (and `decompose()` itself flip-flops run to run); mesh-genus and solid-genus likewise diverge on self-touching meshes (benchy had to build `voxelGenus.mjs` mid-task to cope). A headless "count it all three/four ways and explain the disagreement" tool would cut debug loops that currently cost ~8 bake round-trips down to ~2. Raised by multipart-kits-overview and benchy-foreign-mesh.
- [ ] **Repo-wide Playwright readiness-poll helper** (e.g. `waitForEngineReady(page)`) to replace ad-hoc fixed `waitForTimeout`/modal-race patterns — 3mf-packing-strategy traced two known flaky shards (#912, #911) to exactly this pattern.
- [ ] **Durable-by-default inverse-CAD subagent workspaces.** Converged candidates currently live in gitignored `.plans/` on an ephemeral container and need an explicit copy-and-commit rescue at the end of a session; a crash before that point loses everything but the chat transcript. Raised by inverse-cad-v2.
- [ ] **Stop-hook suppression during long background-agent waits.** The dirty-tree `Stop` hook nudge fired ~20 times during one session's long inverse-CAD agent waits with nothing to reconcile each time. Raised by armor-genericity — needs a hook change, out of scope for a direct docs diff.

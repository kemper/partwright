## What's working (preserve these)

- **The inverse-CAD PLAYBOOK ratchet** (merging each subagent's returned tactics before launching the next wave) keeps compounding across geometry classes it wasn't designed for: the v2 rebuild's thigh→shin→forearm turn-count collapse, the armor phase going 13-for-13 with agents reporting "no new tactics needed," and the 3DBenchy foreign-mesh convergence transferring the armor-era recipe verbatim to a never-seen internet mesh. This is the single most-repeated win across three sessions this week — keep requiring "PLAYBOOK candidates" in subagent return formats.
- **Per-part `expectComponents`/topology gates catching real defects before they ship** — flagged fused articulated pieces, sub-mm debris, and a levelSet vertex-degeneracy in one afternoon (multipart-kits), and a topology MUST gate caught a naive SDF-blending bug that scalar metrics couldn't see (armor phase). Measurement-first, not eyeballing, is paying off exactly as documented.
- **Pre-work exploration before design** continues to collapse scope correctly: reading the seven-location schema ladder and NUL-byte caveat up front made the part-grouping design (an additive field, no schema bump) fall out immediately, and asking the "read-only vs. full editing" design question before polishing the Assembly grid view saved a rebuild.
- **Reducing a new feature to the existing pure/tested core** de-risked two PRs this week: the 3MF/Bambu packing rewrites both isolated the new packing logic as a second pure stage (`packPlates`) alongside the existing grouping/builder code, so the default-behavior refactors landed green on the first unit run.

## Top friction (ranked by cross-session frequency × cost)

1. **[tooling] No warm-browser loop for UI iteration** — 2 sessions (multi-part-assembly-view, 3mf-packing-strategy), both naming it the single biggest wall-clock lever on their task. Every verification round pays a cold ~4s WASM boot + fresh session build inside a new Playwright process, even for pure UI-wiring changes with no geometry work. The repo already solved this for *model* iteration (`partwright daemon start` / `partwright render` / `partwright iterate`) but not for exercising general UI via `window.partwright` + screenshot. **Action: backlog** (new tooling, not a doc fix).
2. **[tooling] Inverse-CAD orchestration has no driver/status tool** — 3 sessions (inverse-cad-v2, armor-genericity, benchy-foreign-mesh). Concretely: a part (`armor_shoulder`) was silently never assigned an agent and only caught by a final manual sweep; there's no way to distinguish "agent still running" from "a best/ exists" so an orchestrator can accidentally bake an interim, still-polishing result; rate-limited/killed background agents needed manual `SendMessage` resume. All three sessions independently asked for the same `status.mjs`-shaped driver (status table + auto-refill + completion detection) that was scoped in the original design but never built. **Action: backlog.**
3. **[tooling] Component/topology counters disagree with each other on degenerate meshes** — 2 sessions (multipart-kits-overview, benchy-foreign-mesh). Engine `decompose()`, the app's stats computation, and a welded-mesh census disagree (and `decompose()` itself flip-flops run to run) on debris-heavy or self-touching meshes; one session had to build a one-off `voxelGenus.mjs` mid-task because mesh-χ genus goes garbage on self-touching meshes (mesh-genus and solid-genus are different quantities and diverge exactly there). **Action: backlog** — a headless "count it three ways and explain the disagreement" tool, generalized to also cover the genus case.
4. **[docs, applied] `src/main.ts` NUL-byte guidance pointed at the wrong tool** — 1 session (3mf-packing-strategy) burned a turn calling the `Grep` tool with `-a` (which doesn't support that flag) before falling back to `Bash grep -an`. Low frequency but a one-line, high-confidence fix. **Action: applied** — CLAUDE.md now says to go straight to `Bash`/`grep -an` for `main.ts` searches instead of trying `Grep` first.
5. **[docs, applied] Worker-client init/ready/error handshake undocumented** — 1 session (multi-part-assembly-view) lost a debugging round to a Worker pool that hung forever (`Promise.all` never resolved, no thrown error) because a client dispatched `execute` before `init`/`ready`, and the resulting `error` message type went unhandled. **Action: applied** — added a "New Worker clients" section to `docs/architecture-notes.md` spelling out the handshake and the two things a new client must do.
6. **[docs, applied] `main.ts` module-scope vs. setup-scope split has no reference** — 1 session (multi-part-assembly-view) wrote new helpers at module scope, then discovered their siblings (`selectPart`, tool-close helpers) live nested inside the setup function, forcing a rewrite into the module-var-assigned-in-setup pattern. **Action: applied** — added a short note to the same `main.ts` CLAUDE.md section describing the two scopes and how to check before writing a new helper.
7. **[docs, applied] Rail/list drag e2e silently misses at the default viewport** — 1 session (part-grouping) burned two runs discovering the parts-list rail is only ~55px tall at 1280×900, scrolling rows out of the drag target. **Action: applied** — added a numbered gotcha to `docs/playwright-guide.md`.
8. **[docs, applied] `showToast` call signature ambiguous from prose alone** — 1 session (mesh-to-code-in-app) guessed `showToast(msg, variant)` instead of the real `showToast(msg, { variant })` and paid a typecheck round-trip. **Action: applied** — added an explicit signature example to the User Messaging section.

## Changes applied in this PR

- `CLAUDE.md` (`src/main.ts` NUL-byte section, renamed to include scope boundaries): clarified that the `Grep` tool itself has no `-a` override (go straight to `Bash grep -an`/`rg -a`), and added a paragraph on the module-scope vs. setup-function-scope split and how to check which a new helper's neighbors use.
- `CLAUDE.md` (User Messaging & the Diagnostic Log): added an explicit `showToast(message, opts)` signature example — options object only, never a bare second argument.
- `docs/architecture-notes.md`: added a "New Worker clients — the init/ready/error handshake" section documenting the `{type:'init'}`→`{type:'ready'}` protocol and the silent-hang failure mode when a client doesn't handle the `error` message type.
- `docs/playwright-guide.md`: added a numbered gotcha for rail/list drag tests needing a taller-than-default viewport.

## Top friction (ranked by frequency × cost)

### 1. [tooling] Catalog goes stale after a shared-builder fix — no drift detection — **~10 sessions**

**Sessions:** `figure-footwear-resize`, `figure-nose-tear-default-lips`, `figure-weld-rebake-regression`, `figure-bust-placement`, `conformal-marking-helpers`, `conforming-panel-garments`, `figure-areola-conformal-offset`, `figure-nipple-flush-rebake`, `figure-nipples-belly-buttons`, `garment-parts-root-cause-and-vestigial-api`, `rebake-figure-catalog` (×2).

**Root cause:** `public/catalog/*.partwright.json` entries are baked snapshots. Fixing a shared builder (`sdfFigure.ts`, a garment helper) changes future runs but leaves already-baked entries silently showing old geometry — caught only when a user notices a stale thumbnail, at which point a fresh re-bake cycle (75s+ per entry, ×20-50 entries) is needed. Several entries also note the catalog has **two sources of truth** (`examples/*.js` vs the baked JSON) with no "rebake everything that changed" command.

→ **Applied:** `CLAUDE.md` model:preview section — added a callout instructing agents to grep the catalog for entries referencing a changed shared builder and re-bake them via the existing (but previously undocumented in CLAUDE.md) `scripts/rebake-figure-catalog.cjs` / `scripts/rebake-shod-figures.cjs`.

→ **Backlog:** A CI or pre-merge check that diffs each catalog entry's stored `code` against the current builder output (stat-diff, not full re-render) and flags drift — tracked informally as `#732` across many sessions. This is the **highest-frequency unresolved backlog item across two consecutive retro reports.**

### 4. [figure-modeling] Thin features fray under a coarse march; genus-rises-with-resolution is a `smoothUnion` artifact, not under-resolution — **5 sessions**

**Sessions:** `figure-hands-fine-mesh`, `figure-accessory-attachment`, `sdf-accessory-layering`, `accessory-attachment-and-eval`, `figure-realistic-feet-toes`.

**Root cause:** A coarse march skeletonizes glasses temples, fingers, chains, and straps; agents independently burned multiple iterations either raising the *global* `edgeLength` (expensive, imprecise) or "fixing" a genus spike by refining further (wrong direction — that symptom means a non-Lipschitz `smoothUnion`/`smin`, not low resolution).

→ **Applied:** `CLAUDE.md` model:preview section — added a callout distinguishing the two failure modes and recommending per-region fine meshing over global resolution bumps.

### 5. [tooling] `model:preview`'s auto-cleanup deletes the "before" reference during A/B comparisons — **3 sessions**

**Sessions:** `figure-nose-realism` (×2 dates), `figure-armpit-shoulder-weld`, `dog-parity-and-colorway-patterns`.

**Root cause:** The stamped-unique-per-run PNG naming (a deliberate anti-stale-cache feature) silently deletes the prior render for the same model file — exactly what an iterative before/after comparison needs to keep.

→ **Applied:** `CLAUDE.md` model:preview section — added guidance to pass explicit distinct `--png` paths when comparing two states of the same file.

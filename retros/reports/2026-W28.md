## Top friction (ranked)

1. **[tooling] No warm-browser loop for general UI iteration** — 2 sessions this week (multi-part-assembly-view, 3mf-packing-strategy), explicitly flagged as a recurring wish from prior retros too. Every UI-touching round pays a cold ~4s WASM boot + fresh session build inside Playwright; a feature iterated ~5 times burns minutes of dead wall-clock. The repo already has this pattern for model rendering (`bin/partwright.mjs daemon start`) but not for general editor/UI interaction. → **Backlog** (needs a new script/harness).
2. **[tooling] No orchestration/convergence driver for large inverse-CAD subagent fleets** — 3 sessions (framework v2, armor genericity, foreign-mesh/Benchy) independently hit gaps in tracking ~15-20 parallel converging agents: a part silently never got assigned an agent (caught only by a final manual sweep), and there's no convention distinguishing "agent finished" from "best/ exists" so an orchestrator can consume a moving target. A `status.mjs` driver was scoped in the original design but never built. → **Backlog** (needs a new script under `scripts/inverse-cad/`).
3. **[docs] `main.ts` NUL-byte trap claimed a 4th session** — this time via a subtler mechanism: the dedicated `Grep` tool has no `-a`/binary-override flag, so trying it first (reasonably, since CLAUDE.md just says "use `grep -a`") wastes a call before falling back to `Bash`. → **Applied**: CLAUDE.md now says explicitly to skip the `Grep` tool for `main.ts` and go straight to `Bash grep -a`/`rg -a`; the "N independent sessions" counter bumped 3→4.
4. **[docs] Rail/list drag e2e specs silently miss at the default viewport** — the parts-list rail is ~55px tall with a few rows at 1280×900, so drag-target rows sit scrolled out of the clipped area and the drag no-ops with no error. Cost two runs + a debug scratch to discover. → **Applied**: added a numbered gotcha to `docs/playwright-guide.md` (sibling to the existing AI-panel viewport note) recommending a taller (~1400px) viewport for rail/list drag specs.
5. **[docs] `showToast` call-signature ambiguity** — CLAUDE.md's messaging section described toast *semantics* but never stated the actual signature, so an agent guessed `showToast(msg, variant)` and paid a typecheck round-trip discovering options are a single object. → **Applied**: CLAUDE.md now states the exact signature inline.
6. **[known, tracked] Tool-history repair has four independent implementations** (Anthropic/OpenAI/Gemini/UI-detector) that can disagree — this week's session burned several turns locating the exact hole and already filed it as **#914**. No new action needed; flagging here so `/issue-reconcile` doesn't lose it.
7. **[tooling] No single trustworthy component counter** — engine `decompose()`, the app's stats computation, and welded-mesh census disagree on degenerate meshes, costing ~8 bake round-trips (~2 min each) to debug by hand in one session. → **Backlog**: a headless "count it three ways and explain disagreements" tool.
8. **[tooling] No headless build-plate packing visualizer** — 3MF plate-layout bugs were verified by parsing `<item>` transforms out of the XML and asserting bed-bounds numerically; there's no glanceable equivalent of `model:preview` for packing arrangements. → **Backlog**.
9. **[tooling] Flaky e2e from fixed `waitForTimeout`/modal-race patterns** — ties to already-filed #911/#912; a shared `waitForEngineReady(page)` readiness-poll helper (or a lint against raw `waitForTimeout`) would retire a class of intermittent shard failures. → **Backlog**.

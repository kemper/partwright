## What's working (preserve these)

- **The bake → `Read` PNG → iterate loop is the MVP.** ~All 10 catalog sub-agents used it. Visual judgment over a rendered thumbnail catches defects — eyes floating above sockets, a hidden gold rim, an illegible sprite — that stats alone would never surface. This is the right primitive for art-directing models.
- **`label(part, 'name')` + `paintByLabels` is excellent for manifold-js and SCAD.** Coordinate-free, survives booleans, `byLabel` descriptors keep saved files tiny. `colors.md`'s "labelled construction" recipe worked first-try repeatedly across 6 agents.
- **`v.mirror('x')` + a 3-tone height-based shade palette gives blocky voxel models real depth cheaply.** A reliable, low-cost idiom for symmetric characters.

## Top friction (ranked)

1. **[catalog / docs] Catalog thumbnail orientation undocumented — 8 agents hit it**
   Root cause: the catalog 3/4 tile camera looks toward +X/+Y (iso azimuth ≈135°, the +X/−Y corner), but this is stated nowhere in `voxel.md`, `colors.md`, or `catalog/README.md`. The natural guess is +Y = front; authors repeatedly baked the back of a character's head and had to rebuild. Cost: 1–2 wasted bakes per agent across ~8 of the 19 models authored over the two passes.
   → **Applied:** one-line Gotcha in `public/ai/voxel.md`; Notes entry in `public/catalog/README.md`.

2. **[catalog / docs] Coordinate paint bloats saved catalog files — 3 agents**
   Root cause: `paintInBox` / `paintNear` / `paintFaces` serialize per-triangle ID lists; one SDF vase reached 17 MB (norm ~250 KB). No documented file-size budget, and `/catalog` eagerly fetches every entry on page load. Switching to `byLabel` dropped the same vase to 185 KB (92×). The replicad robot (no label option after `fuseAll`) still lands at 446 KB — the largest in the catalog.
   → **Applied:** "Catalog entries: prefer `byLabel`" note in `public/ai/colors.md`; file-size Note in `public/catalog/README.md`.

3. **[catalog / docs] Near-white / desaturated colors wash out under flat catalog tile shading — 3 agents**
   Root cause: the tile uses flat/ambient shading that collapses low-saturation colors to gray. No guidance in `colors.md`.
   → **Applied:** "Catalog thumbnail lighting" note in `public/ai/colors.md`.

4. **[voxel / docs] `.smooth({detail:N})` worsens small voxel features — 1 agent**
   Root cause: the existing docs say `detail` gives "finer, more controlled rounding on coarse/small models," which is only half-true — on small dense features (face pixels) it *adds* stair-stepping. One-off, but the docs were actively misleading.
   → **Applied:** caveat added to the `detail` parameter description in `public/ai/voxel.md`.

5. **[replicad / docs] `fuseAll` scrambles `byLabel` with downstream consequences — 1 agent**
   Root cause: `replicad.md` warns that labels scramble, but doesn't explain: (a) every feature must then be targeted by world coordinates, (b) the follow-up entry confirmed the fix is `findFaces({box, normal})` → `paintFaces(triangleIds)`, and (c) `coverageMode:'fully_inside'` has a small-feature cliff (no triangle fits inside → paints zero and errors silently). One-off, but item (c) is a novel finding.
   → **Not applied** (existing `replicad.md` gotcha #9 already covers the label-scramble root cause; the `coverageMode` cliff is addressed in `colors.md`'s fan-topology bleed section). Noted here for awareness.

## Backlog (needs a human or the tooling/linter agent)

- [ ] **`--thumb-azimuth` / `--thumb-elevation` flag on the bake/export** — raised by 8+ agents across both passes. A per-session thumbnail-camera pin would let character and faced models skip baking orientation into geometry entirely. The iso-135 constraint would become advisory rather than mandatory. High value: this single tool would have removed friction from ~8 of the 19 models authored.
- [ ] **Warm persistent bake engine** — re-using a live Chromium + WASM session across multiple re-bakes would roughly halve wall-clock art-direction time (~10–15 s cold boot per manifold bake; ~90 s for replicad/OCCT). Raised once; worth tracking if catalog volume grows.
- [ ] **Richer `BAKE_RESULT` stats** — per-region descriptor kind + resolved triangle count (to confirm `byLabel` and catch a label consumed to zero triangles by a later boolean), plus `voxelCount` for voxel models. Several agents wrote their own inspection scripts to recover this; surfacing it from `runAndSave` directly would remove the need.
- [ ] **Catalog file-size CI gate** — a `lint:catalog` script that reads every `public/catalog/*.partwright.json` and fails if any entry exceeds ~500 KB (or some agreed budget). Raised by 3 agents; prevents the 17 MB regression class from landing silently.

## One-offs noted (no action this week)

- **`smoothUnion` does not propagate inner labels** (outer label wins) — multi-band vertical gradients need a hard-union of labeled solids. Already consistent with existing `manifold-api.md` label-propagation documentation.
- **`.bend(rate, axis)` can't curl a vertical strand forward** — its rotation plane must contain the driving axis. Chaining `capsule()` segments along a parametric polyline is the workaround. Niche geometry finding; too narrow for a general doc entry.
- **`label()` on a degenerate/empty manifold fails late** with `asOriginal() did not produce a valid originalID`. The root cause is the empty/inverted-bounds shape; the error message could be clearer but fixing it requires an engine-level change (backlog for engine layer).

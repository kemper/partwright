## Top friction (ranked)

### 1. [tooling / docs] `bounds()` is conservative ‚Äî surface assertions fail (#672, #673) ‚Äî 2 agents

**Root cause:** `SdfNode.bounds()` returns a loose conservative AABB; the real surface (especially after `smoothUnion`) can be significantly inside or outside it. Two independent figure sessions burned 3‚Äì4 turns each writing test assertions against `bounds()` that failed by 1‚Äì2 units. The existing CLAUDE.md note says `bounds()` is loose in passing, but doesn't prohibit asserting against it.

‚Üí **Applied:** Extended the `smoothUnion` measurement note in CLAUDE.md to say explicitly "never write test assertions against `bounds()` output; use `evaluate(x,y,z)` to find the actual surface."

---

### 2. [tooling] Headless `componentCount` under-reports vs browser bake for thin features (#641) ‚Äî 1 agent, HIGH cost

**Root cause:** The Node mesher bridges small gaps that the browser WebGL-threaded mesher leaves as separate islands. A figure with thin/proud features (cords, braids) returned `componentCount: 1` headlessly but 2‚Äì4 in the browser bake ‚Äî 3 xvfb bake cycles (~75s each) wasted chasing the discrepancy. Nothing in CLAUDE.md warned about this divergence.

‚Üí **Applied:** Added a callout in the `model:preview` section: "Headless `componentCount` can under-report vs the browser bake for near-threshold thin features ‚Äî verify catalog entries in the browser bake."

---

### 3. [tooling / docs] Before/after PNG auto-cleanup with no save path (#673 √ó2) ‚Äî 2 agents (same session, two rounds)

**Root cause:** `model:preview` cleans old per-model PNG stamps at the start of each run. A "before" PNG vanishes once the next render runs on the same model. Both rounds of the figure-nose session hit this independently when trying to build before/after montages.

‚Üí **Applied:** Added a note to the PNG stamp description in CLAUDE.md: "To keep a before/after pair, copy the baseline PNG to a stable filename BEFORE re-rendering."

### 5. [ci] Bot-opened PRs don't trigger `pr-checks` (#641) ‚Äî 1 agent

**Root cause:** PRs opened via MCP under a bot-attributed remote + bot-authenticated git push are suppressed from triggering the `pull_request` event in GitHub Actions. The agent had to substitute local e2e runs (25/25 green) as a substitute. No `workflow_dispatch` exists on `pr-checks.yml` as an escape hatch.

‚Üí **Backlog:** Add `workflow_dispatch` to `pr-checks.yml` so agents can manually trigger it on a PR. Alternatively, document the workaround: "reopen the PR from a human account to kick CI." Raised by 1 agent; cost is HIGH when e2e is the primary validation gate.

### 6. [tooling] Figure smoke bake ‚Äî manifold + paint-label + genus in one headless call (#668, #673 √ó2) ‚Äî 3 agents (highest-frequency new ask)

**Root cause:** Several whole classes of figure bug (buried-bubble voids, 0-triangle paint labels from weld burial, carve genus spikes) are only detectable by ad-hoc probes or the slow xvfb bake (~75s). Three independent sessions independently asked for a one-call "figure smoke" check: manifold + `componentCount`/`voxelPieceCount` sanity + paint-label‚Üítriangle-count + genus, without the full catalog bake.

‚Üí **Backlog:** Design a `npm run figure:smoke -- <file.js>` command (or `model:preview` extension) that: (1) runs the SDF headlessly, (2) reports `isManifold`/genus/component, (3) resolves all `api.paint.label()` calls and reports 0-triangle labels as warnings. The paint-label resolution is the hard part ‚Äî `api.paint.label()` already resolves headlessly (confirmed in #667 round 2). Raised by **3 agents**; highest-frequency new backlog item this cycle.

### 7. [tooling] Catalog dual source-of-truth with no one-shot rebake command (#667, #613) ‚Äî 2 agents

**Root cause:** Catalog entries have two sources of truth: `examples/*.js` source and baked `public/catalog/*.partwright.json`. The manifest references only the baked JSON. A source edit is invisible until each entry is rebaked by hand (`xvfb-run -a node scripts/build-catalog-entry.cjs ...`). Both agents baked entries one at a time, retyping the full incantation each time.

‚Üí **Backlog:** `npm run catalog:rebake -- <id...>` (or `--changed`): reads `bake-manifest.json` (source + gates + palette per entry) and re-bakes the named entries, or all entries whose source `.js` is newer than their `.partwright.json`. The palette/gate declarations already exist ‚Äî just the invocation needs wrapping. Raised by **2 agents**; partially overlaps the `catalog/bake-manifest.json` item from W25. W25's item was the data structure; this is the CLI wrapper that uses it.

### 8. [ci / workflow] Cross-branch feature-collision detection (#590, #606) ‚Äî 2 agents (also raised in W25)

**Root cause:** Two separate PRs built competing `knurl` implementations with divergent parameter vocabulary (pitch/aspect/pattern vs cellWidth/cellHeight/style/sharpness). Both invested significantly before either knew the other existed. The same drift (main advancing mid-task) happened again in #606 but was caught earlier. No mechanism flags "another open branch is extending the same registry."

‚Üí **Backlog (reiterated from W25):** A daily/gate job diffing open PR branches' new exported symbols or `SurfaceOpId`-style enum additions and posting a comment when two branches extend the same registry. Raised by **2 agents** this cycle; was already in W25 backlog. Escalating priority.

---

### 9. [tooling] `api.surface.*` ops invisible in headless preview (#641-catalog, #606) ‚Äî 2 agents

**Root cause:** `model:preview` shades by face normal; `api.surface.*` texture ops are entirely invisible. Agents iterating on texture params work blind and must mentally project the in-browser result. Confirmed by two agents; the catalog-redo session called it "the single highest-value addition."

‚Üí **Backlog:** `model:preview --bake-textures`: run the `api.surface.*` chain headlessly and apply it to the preview PNG. Even at low quality (no WebGPU, single-thread) it would transform texture iteration. Platform-level tooling work; significant but high-ROI.

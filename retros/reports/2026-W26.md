## Top Clusters by Cross-Session Vote

### 1. model:preview paint gap / `--require-labels` gate (≥15 sessions)
The single most-raised issue across all entries. Agents repeatedly shipped figure builds where one or more labels (eyes, iris, pupil, nipple, belly button) painted 0 triangles — invisible in `model:preview`'s normal-shaded render, only visible after the slow catalog bake. `figure:smoke` and `--require-labels` exist and were added precisely for this, but agents kept rediscovering the gap instead of routinely gating on it.

**Status:** Already documented in CLAUDE.md (`figure:smoke` / `--require-labels` section). Re-emphasis applied in Edit 1 (PNG stamp warning keeps paint QC top-of-mind alongside preview).

### 2. Catalog freshness / batch rebake workflow (≥12 sessions)
Catalog thumbnails and entries going stale after figure API changes; batch rebake scripts (`catalog-regen.cjs`, `rebake-figure-catalog.cjs`) exist but agents either didn't know to run them or ran them incorrectly. Two sub-issues: (a) requiring a live dev server on :5173 (already documented), (b) **fresh Playwright page per entry** — reusing one page tears down `window.partwright` after the first bake.

**CLAUDE.md Edit 3 applied:** Added the fresh-page rule to the `build-catalog-entry.cjs` callout.

### 3. componentCount under-reporting in headless/SSR (≥10 sessions)
`model:preview` (Node SSR) under-reports component count for thin features near the meshing threshold — features that survive in the browser GPU path split or merge differently in SSR. Agents trusted the headless count and shipped figures with wrong component splits. `figure:smoke` is already noted as only reliable for *paint resolution*, not component splits.

**Status:** Already documented. No new edit needed — existing callout is clear.

### 4. Investigate/explore before implementing (≥10 sessions)
Agents repeatedly re-implemented capabilities that already existed (projection engine, console methods, reference pipelines) because they didn't run an explore pass first. The parallel-explore pattern (3 agents fanning out over the codebase) consistently returned a complete picture in one round.

**Status:** Already in CLAUDE.md (explore agent, parallel fan-out guidance). The retros affirm the pattern; no new text added.

### 5. `bounds()` / `smoothUnion` / `smin` non-Lipschitz fields (≥8 sessions)
Three related issues raised repeatedly: (a) regression tests asserting exact `bounds()` extents fail on offset/smoothUnion shapes because bounds() is a conservative AABB; (b) `smoothUnion`/`smin` accumulates phantom topology that worsens with mesh resolution — genus rising as edgeLength shrinks is the diagnostic signature; (c) thin SDF features meshed in a rotated frame produce pose-dependent webbing.

**CLAUDE.md Edit 2 applied** (in prior session): Extended the "Measure geometry empirically" callout with all three sub-points.

### 6. Headless bake misdiagnosis — `xvfb` vs dev server (≥8 sessions)
At least 3 independent sessions concluded that catalog baking "can't work in this container" after `xvfb-run` still printed "API never appeared". The real issue was always a missing dev server on :5173. This was already documented in the `build-catalog-entry.cjs` callout.

**Status:** Already documented. No edit added (Edit 3 extended the same callout).

### 7. PNG stamp cleanup — before/after comparisons lost (≥7 sessions)
`model:preview` stamps each render with a unique timestamp suffix and auto-cleans older stamps for the same model file. Agents lost "before" renders when they re-ran preview to check a fix. The solution is to copy the baseline PNG to a fixed path before re-rendering.

**CLAUDE.md Edit 1 applied** (in prior session): Added the "copy to fixed path before re-rendering" note to the stamp callout.

### 8. Part/color lifecycle in multi-part sessions (≥7 sessions)
Color values lost when switching active parts; export bugs where multi-part colors weren't preserved in STL/GLB/3MF. Multiple separate sessions hit this and the root causes varied (IndexedDB transaction ordering, part-switch color-loss on re-run, export pipeline).

**Status:** Logged as a recurring bug pattern; specific fixes went into their respective PRs. No general CLAUDE.md guidance added (the fixes are code-level, not process-level).

### 9. `work-reviewer` as pre-PR gate (≥6 sessions)
Several retros noted the `work-reviewer` caught real issues that typecheck, preflight, and the e2e suite all missed: dropped functionality in a merge, unstamped `addedAt` fields, import-preview readers still pointing at renamed fields. The consensus is it earns its keep on every non-trivial PR.

**Status:** Already positioned in CLAUDE.md as "the right gate before marking a PR ready." No new text added; pattern is confirmed.

### 10. `isolation: "worktree"` branches from `origin/main`, not current HEAD (≥6 sessions)
Agents calling subagents with `isolation: "worktree"` expected the worktree to continue from the caller's feature branch but got `origin/main` instead. This produced diverged branches and confusing diffs.

**CLAUDE.md Edit 4 applied:** Extended the "Git is single-writer" bullet to note the `origin/main` base behavior and the `branch` arg workaround.

### 11. Nested `claude -p` inside repo triggers Stop hook (≥5 sessions)
The figure-eval-loop retro documented this most clearly: shelling `claude -p` from within the repo directory causes the project's Stop hook to fire and replace the model's answer with reconcile text. Three subsequent sessions re-hit the same trap.

**CLAUDE.md Edit 6 applied:** Added a new "Nested `claude -p` must run outside the repo directory" bullet to the Agent Tooling section.

### 12. Issue hygiene — close promptly when resolved (≥5 sessions)
Multiple retros noted the inverse of the "partial implementation never closes silently" rule: issues were left open after they were fully resolved because agents defaulted to caution and didn't close them. The asymmetry (prompt close of partial issues ≠ prompt close of completed ones) means the issue tracker drifts toward false-open entries.

**CLAUDE.md Edit 5 applied:** Extended item 2 in the issue hygiene section to explicitly state the prompt-close-when-done obligation.

### 13. `send_later` unavailable in web/remote sessions (≥4 sessions)
Multiple retros longed for a first-class "notify on CI success" wake-up. In web/remote sessions `send_later` is unavailable and CI-success is not delivered as a webhook, so the watch-the-PR loop requires a Monitor-based background poll.

**Status:** Already documented in the "After Opening a PR" section ("use a Monitor-based background poll"). No new edit added.

### 14. Batch bakes must use fresh Playwright page per entry (≥4 sessions)
`catalog-regen.cjs` and `rebake-figure-catalog.cjs` already do this correctly, but agents writing new bake loops re-hit the issue: reusing one page across `createSession` calls tears down `window.partwright` after the first bake.

**CLAUDE.md Edit 3 applied** (above, cluster 2).

### 15. Prototype-first for subjective/aesthetic work (≥4 sessions)
Build 2–3 throwaway variations, render side-by-side, let user pick direction. Already in CLAUDE.md; the retros confirm it cuts iteration dramatically (the footwear sole example is the canonical one).

**Status:** Already documented. No edit.

## Backlog Items (not auto-applied — need tooling/structural changes)

### BL-1: `partwright` method → AI tool lint
**Raised by:** ≥3 sessions (paintImage retro, reference-images retro, codebase audit)  
**Ask:** A lint rule (or knip-style check) that flags any `window.partwright.*` method absent from `ALL_TOOLS` in `src/ai/tools.ts`, with an explicit allowlist for deliberate omissions. Currently the UI↔API parity gap is invisible until an agent fails in the field.  
**Effort:** Medium (ast-grep rule + allowlist file). Would close the most-cited silent drift vector.

### BL-2: `__testables__` export for clip internals
**Raised by:** 2 sessions (streak-sliver retro, figure geometry work)  
**Ask:** Thin `__testables__` export from `clipByField`, `buildRefinedMesh`, and `subdivideSelected` so unit tests can assert directly on clip output without going through `buildStrokeMesh`. Also a shared `triAspect`/`isSliver` test utility.  
**Effort:** Small (export + test helper). High value for future tessellation work.

### BL-3: `npm run snap` — `--no-tour` / `--run <file>` flags
**Raised by:** 2 sessions (studio-viewport retro, viewport UI work)  
**Ask:** The `snap` helper doesn't suppress the onboarding tour overlay or run arbitrary code before screenshotting. A `--no-tour` flag (sets `localStorage.partwright-tour-completed = 1`) and `--run <file>` flag (executes a code string before capture) would make it usable for UI regression screenshots without writing a throwaway spec.  
**Effort:** Small (extend `scripts/snap.mjs`).

### BL-4: `partwright render --paint` headless path
**Raised by:** 2 sessions (streak-sliver retro, figure paint QC)  
**Ask:** `model:preview` renders model geometry; paint regions live in the app/session layer and can't be rendered headlessly. A `partwright render --paint` path (through the warm-browser daemon) would let CI screenshot the painted result without a Playwright spec.  
**Effort:** Medium-large (daemon integration). Already referenced in `docs/headless-cli.md`.

### BL-5: Capability registry — single source for command palette + API
**Raised by:** ≥4 sessions (paintImage, reference-images, codebase-audit, character-creator retros)  
**Ask:** A single capability registry that both the command palette and `window.partwright.help()` derive from. Currently the drift between UI affordances and API methods is structural — there's no typed link between a DOM button and a `window.partwright` method.  
**Effort:** Large (architectural refactor). Already noted as a deferred refactor in CLAUDE.md. This backlog item is a reminder that it remains the right long-term fix for BL-1 above.

### BL-6: `explore` agent — `.claude/agents/explore.md` update
**Raised by:** Multiple sessions  
**Ask:** The explore agent's instructions could explicitly note the `mcp__serena__*` tool preference order (Serena for definition/reference queries, ripgrep for string literals) and the pattern of running 3 parallel fan-out agents for codebase-wide questions. The current instructions are correct but don't foreground the parallel pattern.  
**Scope:** `.claude/agents/explore.md` — excluded from auto-apply per retro-review rules.

### BL-7: `work-reviewer` — add "was this issue fully satisfied?" check
**Raised by:** 2 sessions  
**Ask:** The `work-reviewer` currently checks code correctness, back-compat, security, and UI consistency. Adding a pass that reads the PR's linked issues (via GitHub MCP) and checks whether the diff actually satisfies their acceptance criteria would automate the issue-hygiene close-out check.  
**Scope:** `.claude/agents/work-reviewer.md` — excluded from auto-apply.

### BL-8: Monitor-based CI poll helper / `send_later` substitute
**Raised by:** ≥4 sessions  
**Ask:** A documented snippet for the "arm a Monitor background poll to re-check PR CI status after N minutes" pattern, since `send_later` is unavailable in web/remote sessions and CI-success is not a webhook. Currently each session re-invents the pattern.  
**Effort:** Small (add to `docs/playwright-guide.md` or a new `docs/ci-watch.md`).

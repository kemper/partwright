## What's working (preserve these)

- **`model:preview` + `sharp` crops at 1200+ px** — the workhorse across every figure session. Catching defects invisible at the default tile size, multi-angle `--view "az,el;…"` tiles, and native crops were cited by almost every entry.
- **`model-sculpt` subagent delegation** — keeping dozens of preview PNGs out of the main context. 10 parallel subagents on the 20-figure batch sessions worked; the image-token discipline in CLAUDE.md paid off.
- **`figure:smoke --require-labels`** — the 2s headless paint-QC gate added in W25 caught buried-eye/areola failures before the 75s xvfb bake in several sessions. The tooling is working; the remaining gap is *colored* preview.
- **SDF `evaluate()` probe via vite-node** — "measure before guessing" for smoothUnion extent and buried features. Sessions that reached for it first saved 4+ render rounds; the CLAUDE.md note about this is correct and underused.
- **`work-reviewer` on no-op PRs** — the versioned-deploy session specifically noted that the review found base-leak gaps invisible to a base-`/` test suite; correctness review is the only net for latent-at-`/vN/` bugs.
- **Four parallel audit subagents with explicit "FINE" lists** — the FINE lists were as valuable as the defect lists for bounding blast radius.
- **Single-writer discipline for catalog baking** — subagents author+verify only; orchestrator owns all git/bake. Zero working-tree races when respected.
- **`--require-labels` on `build-catalog-entry.cjs`** — caught 8/20 defects that `model:preview` reported clean.
- **Bisect-by-removal for componentCount splits** — fast fix for "browser says 2, SSR says 1" by removing parts one at a time from the bake.

## Top friction (ranked)

### 1. [tooling] No headless colored render — `model:preview --palette` gap — ~20 independent sessions

**Sessions:** figure-api-showcase-catalog (×2), figure-catalog-batch-pr692, figure-twenty-fullbody-catalog-pr690, figure-builder-diversity, figure-api-defect-fixes, figure-areola-backfoot, figure-belly-knob-bake-dx, figure-footwear-plantarflexion, figure-bust-placement, figure-nipple-generic-vs-local, figure-eyebrow-system, runway-gown-topless-cone-fix, figure-areola-conformal-offset, figure-nipple-flush-rebake, figure-nipples-belly-buttons, catalog-redo-8-models, figure-muscle-axis, figure-nose-realism, figure-api-ears.

**Root cause:** `model:preview` shades by face normal; it can't show palette label colors. Confirming "is the dress covering the chest?" or "do the eyes actually paint?" requires a full 75s xvfb browser bake with a running dev server. This is the highest-cost per-session friction point across the entire week — nearly every figure or catalog session hits it.

→ **Applied (partial):** `CLAUDE.md` — clarified the colored bake callout with `THUMB_AZIMUTH=0` = front/−Y convention, and added a paint-relief-floor note (flush regions dither; ≳1 triangle of proud relief is the fix).

→ **Backlog (tooling):** `model:preview --palette <file>` flag that applies a label→colour palette to the fast preview PNG. Even low-quality tinted-by-label output would collapse a 75s loop into 2s for the most common paint-verification question. Raised by **~20 agents** — highest-frequency unresolved tooling ask in the entire history of this retro.

### 3. [tooling] Catalog freshness: source and baked `.partwright.json` can silently diverge — 5 sessions

**Sessions:** figure-nipple-flush-rebake, figure-areola-conformal-offset, figure-nipples-belly-buttons, conformal-marking-helpers, figure-bust-placement.

**Root cause:** Editing an engine helper (`sdfFigure.ts`) and the figure examples (`.js`) does NOT update the baked `.partwright.json` catalog entries. 7 of the 9 affected figures aren't in `bake-manifest.json`, so `catalog-regen.cjs` doesn't touch them. The user notices stale nipples/eyes/areolae in the deployed catalog after a merge. There's no CI check that source and bake agree.

→ **Backlog (tooling/CI):** A `catalog:freshness` CI check (or `lint:catalog`) that headlessly re-bakes each figure entry, diffs per-label triangle counts and componentCount against the committed `.partwright.json`, and exits non-zero on divergence. Would convert "user notices stale catalog" into "PR-checks fails." Raised by **5 agents**; second-highest-frequency backlog item. Companion: add all hand-baked figure entries to `bake-manifest.json` so `catalog-regen.cjs` owns them.

### 4. [tooling] `model:preview` (Node SSR) vs browser bake `componentCount` divergence — 5 sessions

→ **Backlog:** A browser-faithful headless component check (the same Chromium mesher, headless, querying `componentCount`) so the catalog bake is the only place a component split surfaces. Raised by **5 agents**.

### 5. [docs] `--view az,el` azimuth convention — CLI rasterizer ≠ `multiview.ts` — 4 sessions

**Sessions:** figure-eyebrow-system, figure-nipple-generic-vs-local, figure-nipples-belly-buttons, figure-areola-backfoot.

**Root cause:** `model:preview --view 0,el` is NOT the front face; the CLI rasterizer uses a different azimuth origin than `multiview.ts` (0=front/−Y). `THUMB_AZIMUTH=180` gives the back. Named views (`--views front`) are the reliable path. Multiple agents burned bakes discovering this. The W25 backlog noted the ai.md doc fix as a backlog item; the CLI-vs-browser convention confusion is the concrete agent pain.

→ **Applied:** `CLAUDE.md` model:preview colored-bake callout — added `THUMB_AZIMUTH=0` = front/−Y; added note that `--view 0,el` is not reliably "front" and named `--views front` is the safe choice.

### 6. [docs] `bounds()` is a loose AABB, not the true surface extent — 3 sessions

**Sessions:** figure-realistic-feet-toes, figure-nose-realism (implicitly), figure-api-defect-fixes.

**Root cause:** `SdfNode.bounds()` returns a conservative AABB; assertions against it for surface positions fail even when geometry is correct. The existing CLAUDE.md note said "don't trust analytic extents" and "`bounds()` is loose" but didn't call it out sharply enough.

→ **Applied:** `CLAUDE.md` — strengthened the smoothUnion note to explicitly say `SdfNode.bounds()` always returns a loose conservative AABB and is not the true surface extent; assertions against it will be wrong by the blend halo.

## One-offs noted (no action this week)

- **Compaction / orphaned tool_result** — already fixed in PR #721; the repair now handles both tool_use and tool_result directions. No action needed.
- **`MAX_DETAIL_REGIONS` cap 16→24** — raised to 24 in PR #727 after surfer/tai-chi/lotus failed to render (14 face + 4 foot = 18 > 16). The unit test had a hardcoded `17`; both the cap constant and the test threshold must move together. Fixed and noted.
- **`python json.dump` / `ensure_ascii=True`** — silently re-escapes em-dashes in JSON, turning a 3-entry manifest add into a +121/-91 churn. Reflex: always use `ensure_ascii=False` when editing JSON via Python. One-off process note; not a CLAUDE.md addition.
- **`CLIP/clip condition:` `hemZ + big < zTop`** — finite `big`-tall half-space silently amputated tall dresses; use content-extent-derived bounds for "infinite" clips. Fixed in PR #742.
- **Guess issue numbers before filing** — two sessions guessed an issue number in code comments before filing and had to rewrite them. Process reflex: file the issue first, get the number, then reference it. One-off; existing "file first" hygiene in CLAUDE.md is sufficient.
- **Promptlog guard staging race** — the promptlog hook fires before `git add` when Write + git-add run in parallel; the log file exists but isn't staged. Run `git add <promptlog>` first, then commit. One-off; not a doc change.
- **Worktree `isolation:` + catalog-only commits skip pr-checks** — commits touching only `examples/*.js` + `public/catalog/*.json` don't dispatch pr-checks (path-ignore mismatch). Run `catalog.spec.ts` locally for those commits. One-off.
- **Pixel-classifying colored bake to distinguish skin vs garment** — useful pattern for objective "is that skin?" in coverage bugs; worth noting but not a CLAUDE.md rule.

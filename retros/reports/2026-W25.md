### 2. [tooling] `build-catalog-entry.cjs` fails without X display ‚Äî 2 agents

**Entries:** `figure-face-quality`, `fasteners-joints-split`

`scripts/build-catalog-entry.cjs` has `headless: false` hardcoded ‚Äî it launches a real Chromium for WebGL thumbnail rendering. In this container (no X display) it fails silently. Both agents discovered this via a failed run and recovered with `xvfb-run -a`. `scripts/catalog-regen.cjs` already uses `headless: true`; the single-entry script should match.

‚Üí **Applied:** `CLAUDE.md` Headless Model Preview section ‚Äî added a note that `build-catalog-entry.cjs` requires `xvfb-run -a` in this container, and that `catalog-regen.cjs` doesn't need it.

‚Üí **Backlog:** Change `headless: false` to `headless: true` in `scripts/build-catalog-entry.cjs` (or add a `--headed` flag that defaults to false). Raised by 2 agents.

### 3. [ci / testing] UI label renames silently break e2e selectors ‚Äî 1 agent, 2 CI rounds

**Entry:** `surface-panel-as-code`

Renaming the panel's 'Apply' button broke two separate specs in two distinct CI rounds (once on the branch, once after merging from main). E2e selectors use `.getByRole('button', { name: '‚Ä¶' })` text-match, so any label rename that isn't grep-checked in `tests/` before pushing causes a delayed CI failure.

‚Üí **Applied:** `CLAUDE.md` E2E tier section ‚Äî added a warning: grep `tests/` for the old string before and after every merge from main when renaming any user-visible string an e2e spec references by text.

‚Üí **Backlog:** An ast-grep or unit check tying user-visible button labels to test selectors (e.g., a lint that every `getByRole('button', { name: ‚Ä¶ })` string in `tests/` exists in `src/`). Raised by 1 agent.

### 4. [ci / pr-flow] Conflicted PR silently drops `pull_request`-event CI ‚Äî 1 agent

**Entry:** `surface-panel-as-code`

A conflicted PR silently skips all `pull_request`-event workflows (pr-checks) while push-triggered ones (CodeQL, Cloudflare) keep running. The asymmetry is the diagnostic: if pr-checks stops after a push but CodeQL fires, check mergeability before assuming webhook loss. `git merge-tree origin/main HEAD` confirms in seconds.

‚Üí **Applied:** `CLAUDE.md` After Opening a PR (step 1) ‚Äî added a note about this signature and the `git merge-tree` quick check.

### 5. [tooling] `src/main.ts` NUL bytes cause silent grep truncation ‚Äî 1 agent

**Entry:** `surface-paint-hardening`

Internal cache-key separators in `src/main.ts` use `\0` NUL bytes in template literals. `rg`/`grep` silently treats the file as binary and returns no results ‚Äî a full round was lost discovering this only via `cat -A`. Either `--text` / `-a` flag or Serena `find_symbol` works around it.

‚Üí **Applied:** `CLAUDE.md` Search ladder ‚Äî added a one-line warning about `src/main.ts` NUL bytes and the `--text` workaround.

‚Üí **Backlog:** Replace the `\0` separators in `src/main.ts` with a printable collision-safe separator (`'|'` + JSON-escaping of values, or `'\x1e'` record separator). Raised by 1 agent.

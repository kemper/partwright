## Top friction (ranked)

### 1. [tooling] `src/main.ts` NUL bytes break grep/Edit ‚Äî 3 independent sessions

**Sessions:** `surface-paint-hardening`, `surface-worker-auto-apply`, implicitly `surface-texture-persistence` (via Edit failure).

**Root cause:** `src/main.ts` embeds literal `\0` bytes as template-literal cache key separators (`surfaceBaseKey`). `grep`/`rg` treats the 14k-line file as binary and silently truncates results. `Edit` and perl regex fail at the NUL boundary. Each session spent 4+ turns rediscovering this independently.

‚Üí **Applied:** `CLAUDE.md` ‚Äî added "Editing `src/main.ts` ‚Äî NUL-byte zones" subsection under Development Guidelines, with the `grep -a` / Python-slice workarounds.

### 2. [tooling / catalog] `build-catalog-entry.cjs` requires `xvfb-run` in CI/container ‚Äî 4 sessions

**Sessions:** `figure-face-quality`, `fasteners-joints-split`, `figure-audit-fixes-rocker`, `figure-loomis-clothing-naming-rebake`.

**Root cause:** `scripts/build-catalog-entry.cjs` hardcodes `headless: false` (uses headed Chromium for real WebGL thumbnails). Every container/remote session re-derives the `xvfb-run -a node ...` incantation from scratch. `catalog-regen.cjs` already uses `headless: true`.

‚Üí **Applied:** `CLAUDE.md` model:preview section ‚Äî added a callout block with the `xvfb-run` incantation and a note that `catalog-regen.cjs` behaves differently.

‚Üí **Backlog:** Fix `build-catalog-entry.cjs` to detect `$DISPLAY`/headless capability and either auto-wrap `xvfb-run` or exit with a clear error and the wrapper suggestion. Raised by 4 agents; a one-line change to the script removes the recurring friction.

### 3. [docs / workflow] `send_later` unavailable in web/remote sessions ‚Äî 3 sessions

**Sessions:** `surface-texture-persistence`, `agent-efficiency-tooling`, `figure-loomis-clothing-naming-rebake`.

**Root cause:** The PR-watch playbook (CLAUDE.md "After Opening a PR") mentions `send_later` for a ~1h self check-in to catch CI-success / main-moved / merge-conflict transitions. But `send_later` isn't available in web/remote sessions. All three sessions went dark after the last push; the human had to confirm green status.

‚Üí **Applied:** `CLAUDE.md` "After Opening a PR" step 1 ‚Äî added explicit note that `send_later` is unavailable in web/remote sessions and to use a Monitor-based background poll instead.

‚Üí **Backlog:** Expose `send_later` (or a minimal equivalent) in web/remote session contexts. Platform-level; filed for awareness.

### 5. [docs / schema] Schema-bump checklist gap ‚Äî 1 session (full CI re-run cost)

**Session:** `surface-texture-persistence`.

**Root cause:** Bumping `SCHEMA_VERSION` to 1.14 involved 7 synchronized locations; the agent missed the test that hardcoded `'1.13'` and only caught it in the CI shard (cost: one full re-run round). `dbSaveVersion` also has a 16-arg positional signature that required `null, null` placeholders for new trailing params.

‚Üí **Applied:** `CLAUDE.md` ‚Äî added "Session Schema Migrations" subsection after IndexedDB Transactions, listing the 7 locations that must stay in sync and noting the `SCHEMA_VERSION` constant import pattern.

‚Üí **Backlog:** Convert `dbSaveVersion`'s trailing positional args to an options object. Raised once; low urgency until the next schema field lands.

### 6. [docs] `model:preview -p` silently ignored without `paramsSchema` ‚Äî 1 session

**Session:** `figure-loomis-clothing-naming-rebake`.

**Root cause:** `-p key=val` on `model:preview` only binds when the snippet declares `api.params = { ... }`. Without a `paramsSchema` in the stat block, the override is silently dropped ‚Äî identical renders on every pass. The agent detected it only by noticing identical file sizes (~15 min and a fallback to hardcoded variant files).

‚Üí **Applied:** `CLAUDE.md` model:preview section ‚Äî added a callout noting the `paramsSchema` requirement and the "identical triCount/bbox = probably missing paramsSchema" diagnostic.

**Entries reviewed:** 95 (spanning ~90 distinct sessions/PRs — a handful of files are harvested duplicates of the same effort from unmerged catalog PRs #690/#692/#693, noted below)  ·  **Span:** 2026-06-11 → 2026-06-28

This is a large backlog catch-up — the inbox hadn't been reviewed since `2026-W25` (last archived entry: 2026-06-11 21:45). Given the volume, all 95 entries were read via six parallel digest agents, then clustered by hand across all six digests.

## What's working (preserve these)

- **Explore-agent pipeline-mapping before implementing** is the single most repeated positive pattern in the entire corpus — 20+ sessions independently credit a read-only `explore` (or several parallel `explore`s) tracing the real pipeline *before* any code change for turning a guess into a grounded, low-risk diff, and for redirecting fixes from symptom to root cause. This is already CLAUDE.md doctrine; the data says it's paying for itself constantly. Keep recommending it as the default first move on non-trivial work.
- **Delegating multi-pass visual iteration to `model-sculpt`/subagents** to keep preview PNGs out of the main context is praised repeatedly across figure/catalog/accessory work, and is already documented (`CLAUDE.md`'s "Delegate multi-pass visual iteration…" callout).
- **Measure geometry empirically (`evaluate()` / vite-node SDF probes) instead of eyeballing or trusting `bounds()`** shows up as the decisive move in a dozen bug hunts (nostril placement, buried eyes, foot direction, garment overlap volume). Already documented; kept working every time it was used.
- **Prototype 2–3 variants and let the user pick before wiring into a builder**, for subjective/aesthetic work, converged fast every time it was actually followed (studio viewport, several figure features) and cost real rework the few times it was skipped (grasp API, bump placement).
- **Fan-out of read-only audit agents with cross-agent frequency as the confidence signal** (the post-release codebase audit's 13-agent sweep) is exactly this retro-review's own method, and the data backs it: independently-flagged findings were the real ones.
- **Reproduce visually before *and* after theorizing a fix** (render before touching code, then re-render to confirm) reliably separated real root-causes from plausible-sounding ones.

3. **[process] A capability/API registry linking UI ↔ `window.partwright` ↔ command palette ↔ AI tools is the most-repeated structural wish in the whole corpus** — raised independently in 15+ sessions across completely different feature areas (attachments, character-creator, import, exports, audits, SDF verbs, publishing). CLAUDE.md already names this as a known, deliberately-deferred larger refactor (`UI ↔ JS-API parity` section) — the frequency here is a strong signal it should be scoped and scheduled rather than left as permanently-deferred. Left as a **backlog** item (see below) since it's a genuine structural refactor, not a doc fix.
4. **[verification discipline] Fixing the wrong branch / wrong code path because the actual repro wasn't confirmed first** — 7+ sessions (diva sleeve fixed the unreached `'long'` branch; import-select shipped a fix against the new-session path when the user's repro was the merge path; two distinct "no color" bugs looked identical until the render state was actually checked). **Action:** folded into the same CLAUDE.md callout as #2 — "reproduce their exact repro first" now covers code path/entry point, not just camera angle.
5. **[export/integration] Validating against a synthetic/CLI proxy instead of the real downstream consumer** — 4 sessions, all Bambu/3MF work: a headless slicer CLI validated cleanly while the real GUI crashed on a different load path, three sessions apart, before a user-supplied crash report ended the guessing. **Action:** added a general "validate against the real consumer" bullet to Agent working discipline.
6. **[process] `promptlog-guard.sh` appears to fire "too early" on a compound `git add -A && git commit`** — 2 sessions read this as the guard misfiring rather than a sequencing issue (the guard denies before `add` stages the log). **Action:** added a one-line clarification to the Prompt Logs section.
7. **[agents] Delegated subagent "done" claims, and worktree isolation, aren't automatically trustworthy** — a worktree branched from stale `origin/main` instead of the current feature HEAD (producing an unmergeable rewrite), and a subagent's "validated" claim was wrong on inspection. **Action:** added two bullets to Agent working discipline.
8. **[tooling, no CLAUDE.md action] No CI-success wake in remote/web sessions (`send_later`/`gh` unavailable)** — the single most *frequently* mentioned friction of all (15+ sessions), but it's already fully documented in the "After Opening a PR" section with the Monitor-poll workaround, and the actual fix is a harness capability outside this repo's control. No further doc action; flagged again in the backlog as a standing harness ask in case it's ever prioritized upstream.

## What's working (preserve these)

- **`model:preview` fast loop** — 2-second headless render + JSON stats is the right instrument for tight geometry iteration. Consistently praised across figure, accessory, garment, and catalog sessions.
- **`--require-labels` gate** — caught 8/20 bake defects in one figure-catalog batch session. Worth running before every bake PR.
- **`figure:smoke` for label QC** — genus + 0-triangle label check in ~2 s; reliable first-pass before the expensive colored bake.
- **`vite-node` probe scripts** — empirical SDF probing (walk `evaluate(x,y,z)` to sign flip) is the reliable root-cause technique for smoothUnion geometry; beats guessing from analytic extents.
- **Delegating multi-pass iteration to `model-sculpt`** — keeps image tokens out of main context; multiple sessions praised the pattern.
- **`work-reviewer`** — consistently catches parity gaps, unstamped fields, and blast-radius misses before merge.
- **Parallel subagents with worktree isolation** — praised in figure-catalog batch sessions for clean fan-out; the single-writer rule held.
- **`explore` agent pipeline map** — praised for surface, figure-layering, and import-flow investigations; prevents "code blind" root-cause guessing.

## Top friction (ranked by distinct session count)

### 1. [paint / figure] Flush-surface paint dithers — no rule documented
**~12 sessions** (areola, eyebrow, skin marking, makeup, footwear label seam) · cost: medium per incident

Every decorative region sitting exactly flush on its surrounding surface produces salt-and-pepper edges in the colored bake. `stats.paintOps` reports non-zero triangles (no warning fires), so the tool gives false confidence. The fix is consistent — add ~1 triangle of outward relief — but each session rediscovered it independently.

→ **Action taken:** Added "flush-surface paint dithers" note to the `model:preview` paint bullet in `CLAUDE.md` (line ~222), including the fix and the false-confidence trap.

---

### 2. [figure / accessory / garment] Front-only QC ships embedding defects
**~10 sessions** (belts on arms, necklaces through dresses, garments with back-only coverage holes, shoes embedding body)

Front/iso is the default angle. Side and back views are not in any mandatory checklist. Multiple sessions shipped a "fixed" result that only broke from the side.

→ **Action taken:** Extended the multi-angle callout in `CLAUDE.md` to explicitly name garment/accessory work as requiring side (`--view 90,10`) and back views before sign-off.

---

### 3. [catalog] Catalog `.partwright.json` goes stale silently — no documented rule
**~8 sessions** (figure footwear, figure-weld, figure-catalog-batch, bust-placement, garment-parts, rebake-figure-catalog, diva-sleeve, areola-conformal)

Source-code fixes don't propagate to catalog until an explicit re-bake. Multiple sessions discovered mid-PR that the catalog entry was still showing old geometry. No CI freshness gate exists.

→ **Action taken:** Added "catalog files are a snapshot — they go stale whenever source changes" callout to the `build-catalog-entry.cjs` section in `CLAUDE.md`.

→ **Backlog:** CI catalog-freshness gate (see below).

---

### 4. [tooling] `model:preview` cannot show palette-based paint colors (only in-code `api.paint.*`)
**~10 sessions** (figure-eyebrow, figure-nipples, figure-areola, gown-topless, character-creator-panel, paint-buried-triangle, runway-gown, figure-invariant)

Sessions repeatedly noted "model:preview shades by normal, can't show palette colors" — meaning byLabel palette assignment (catalog figure colorway) is only visible at bake time. In-code `api.paint.*` DOES resolve headlessly, so the tool is not useless for paint, but figures that rely on a palette file aren't colored without `--palette-file`. The existing CLAUDE.md already documents `--palette-file`; the gap is that sessions forget and waste turns.

→ **No new rule added** (already documented under `model:preview renders color by default`). **Backlog:** headless colored render for palette-colored figures without a dev server (i.e., a standalone `--bake-colors` mode).

---

### 5. [geometry / tooling] SSR vs browser `componentCount` divergence rediscovered repeatedly
**~8 sessions** (figure-builder-diversity, figure-catalog-batch, figure-api-defect-fixes, figure-weld-rebake-regression, figure-api-showcase-catalog, figure-layering, figure-hands)

`model:preview` (Node SSR) under-reports `componentCount` for near-threshold thin features (hair coils, proud welds, fine-detail mesh). The browser bake is the authoritative oracle. This is already documented in CLAUDE.md under `figure:smoke`, but sessions keep re-hitting it.

→ **No new rule added** (already documented). **Backlog:** browser-faithful headless componentCount check (see below).

---

### 6. [figure / SDF] SDF occluder layering for accessories has no formal architecture
**~8 sessions** (accessory-attachment, sdf-accessory-layering, belt-on-arm, figure-layering-architecture, garment-parts, figure-grip-flush-band, figure-grasp-api-rebuild, figure-invariant-gate)

Accessories (belts, necklaces, props) geometry-overextends into limbs because there is no layer concept in the figure SDF API. `F.sharedSolid` and `clear` occlusion patterns were developed ad hoc across sessions. Root fix shipped (F.grasp, F.layers concepts), but no formal spec or invariant gate prevents regression.

→ **Backlog:** Figure invariant gate (pillar-2 — automatic build-time overlap/occlusion check).

---

### 7. [tooling] No `npm run catalog:rebake` generic script
**~6 sessions** (rebake-figure-catalog, figure-footwear-resize, figure-areola-conformal, figure-bust-placement, figure-weld-rebake-regression, figure-catalog-batch)

Every session that touched multiple figures had to write or copy a bake loop. The template (auto-discover by `code.includes`, in-place splice, fresh page per entry) is well-understood but not encapsulated.

→ **Backlog:** `npm run catalog:rebake [--changed] [entryId...]` generic script.

### 9. [architecture] Capability registry needed for UI ↔ AI parity
**~5 sessions** (ai-part-target-addressing, codebase-audit-post-release, catalog-split-view-race, versioned-deploy-base-aware, paint-image-ai-tool)

Every session that added a UI feature and an AI tool noted: there is no typed link between them. The `help()` table in `src/main.ts` and `src/ai/tools.ts` can drift silently. A capability registry was proposed repeatedly as the structural fix.

→ **Backlog:** Capability registry (single source for UI, command palette, AI tools, `help()`, docs parity check).

### 10. [paint / geometry] `bounds()` is a loose conservative AABB — multiple sessions re-learned
**~5 sessions** (figure-realistic-feet-toes, figure-shoe-offset-redesign, figure-footwear-plantarflexion, figure-areola-conformal, sdf-accessory-layering)

`bounds()` returns a conservative AABB, not the true surface extent. `smoothUnion` / `.round()` bulge past either input's bounds. This is already documented under "Measure geometry empirically when `smoothUnion` is involved." No new rule needed — it's there.

→ **No action** (already documented).

## Backlog (needs human or tooling agent)

- [ ] **Headless colored render without a dev server** — `model:preview --bake-colors` (or a palette-aware render mode) that applies byLabel palette + bakes paint in one call, without requiring a running `:5173`. Raised by ~10 sessions. Would eliminate the "can't see paint headlessly" half of SSR/browser drift. Estimated impact: high.

- [ ] **Catalog freshness CI gate** — A GitHub Action (or `npm run catalog:lint-freshness`) that re-bakes a sample of entries and diffs them against committed `.partwright.json` / `.thumb.png`. Raised by ~8 sessions. The test should at minimum detect geometry-hash drift (not necessarily pixel-identical thumbnails). Estimated impact: high.

- [ ] **Browser-faithful headless `componentCount`** — A lightweight path that runs the browser meshing pipeline (not Node SSR) in a headless Chromium to get a componentCount that matches the bake oracle. OR: a documented threshold for "if Node SSR says N, browser says N±M". Raised by ~6 sessions. Related to `figure:smoke --fidelity browser`.

- [ ] **Figure invariant gate (pillar 2)** — A build-time check (run in `figure:smoke` or as its own `npm run figure:invariant`) that asserts: no accessory label geometry overlaps with limb labels; layer priority is consistent; occluder coverage ≥ threshold. Raised by ~5 sessions across accessory and garment work. Design sketch in `2026-06-23-figure-invariant-gate.md` and `2026-06-23-figure-layering-architecture.md`.

- [ ] **`npm run catalog:rebake [--changed] [id...]`** — Generalize the per-figure rebake loop (already written ad hoc ~6 times) into a single script: auto-discover entries by builder name, splice in-place, fresh page per entry, verify output. Raised by 6+ sessions.

- [ ] **Capability registry for UI ↔ AI parity** — A single source-of-truth registry that both the command palette and `window.partwright` derive from, with a lint rule that fails if a UI button has no corresponding API method. Raised by ~5 sessions; larger architectural change. Design notes in `2026-06-17-ai-part-target-addressing.md`.

- [ ] **Figure-API machine-readable schema** — Export `figure-api.json` with every enum, preset list, and parameter range. Raised by `2026-06-20-character-creator-panel.md`. Enables tooling (character creator panel auto-populating from schema, eval harness verifying option lists).

- [ ] **Per-figure visual-defect snapshot** — Committed PNGs (or hashes) per catalog figure that CI can diff after a source change. Related to catalog freshness gate but oriented toward regressions (shape change, paint drift) rather than freshness. Raised in `2026-06-17-figure-triage-verify-before-act.md`.

- [ ] **Headless Bambu GUI load validation** — A headless script (separate from CLI validation) that loads a generated `.3mf` into the actual Bambu import path to catch GUI-vs-CLI divergence. Raised by `2026-06-18-bambu-3mf-export.md` and `2026-06-20-bambu-more-printers.md`.

- [ ] **`send_later` self-wake in web/remote sessions** — Platform feature. Current workaround (Monitor poll loop) is fragile and burns context. Raised by ~5 sessions. File as a platform request.

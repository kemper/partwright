Consolidated from **5 `model-sculpt` sub-agents** that each built a catalog
figure with the brand-new `api.sdf.figure` rig (waving kid, double-biceps
strongman, chibi wizard, warrior lunge, ballerina) via the headless
`model:preview` render→`Read`-PNG→adjust loop against printability gates.
Frequency = number of *independent* agents who hit an item. Most asks were
**fixed inside this same PR** and are tagged `[DONE]`; the remaining one is a
tooling follow-up.

## Liked / Worked
- **The rig kills the two worst figure-failure modes structurally** (5/5). Every
  agent got `componentCount: 1` and `isManifold: true` on the first or second
  pass — no floating-part splits, no coordinate guessing. The whole reason the
  API exists, validated.
- **`rig.joints.*` for placing accessories** (wizard, ballerina) — hats, staffs,
  tutus snapped to `crown`/`hand`/`navel` instead of hand-typed coordinates.
- **`headsTall` as the single stylization dial** (5/5) — `3` gave a proper chibi,
  `8` an elegant dancer; the rig rescaled coherently.
- **Composing custom `api.sdf` primitives onto the welded body** (wizard hat,
  ballerina tutu) worked — figure parts and raw SDF mix freely.

## Lacked
- **Raised-arm poses were geometrically unreachable** (2/5 — strongman, ballerina,
  the strongest signal). Elbow flexion curled the forearm *backward* once the arm
  was abducted, so double-biceps / ballet-fifth couldn't put the fist up; the
  `twist` param was parsed but **never wired into the FK**. **[DONE]** Wired
  `arm.twist` as a forearm-roll DOF — `twist≈90` now lifts the fists up.
- **No symmetric pose shorthand** (strongman) — every symmetric pose meant
  copy-pasting `armL`/`armR`. **[DONE]** Added `pose.arms`/`pose.legs` that seed
  both sides (per-side keys override).
- **Feet detached + `F.base` didn't reach for elevated-ankle poses** (warrior;
  latent for ballerina), giving `componentCount > 1`. **[DONE]** Feet now follow
  the ankle, and `F.base` auto-widens to the stance and rises to the lowest foot.
- **Within-limb "balloon-animal" seam at knees/elbows** (kid, warrior, strongman)
  — the two-radius taper showed a midpoint bulge. **[DONE]** Overlapped the taper
  capsules past mid + softer weld k.
- **Garment waist anchor unclear** (ballerina) — hips are leg-insertion points, so
  skirts/tutus cut through the thighs. **[DONE]** Added `rig.r.waist` + documented
  `rig.joints.navel` as the waist; clothing builders note it.
- **Accessory placement offset math** (wizard) — translating a hat to the crown
  meant hand-computing half-heights. **[DONE]** Added `figure.placeAt(node, joint,
  {anchor: center|bottom|top})`.
- **Hidden rig fields undocumented** (strongman, ballerina) — `r.chestX/chestY/
  pelvisX/pelvisY`, the `build` multipliers. **[DONE]** Documented all in
  `figure.md`.

## Learned
- **The elbow needs a roll DOF, not a smarter hinge.** Two agents independently
  proved no `abduct`/`flex`/`elbow` combination lifts a side-arm's fist — the
  curl plane rotates with abduction. Anatomically the missing input is humeral
  rotation; exposing it as `twist` (an existing-but-dead param) was the clean fix.
- **Sub-agent frequency is a strong prioritizer.** The one issue 2+ agents hit
  (raised arms) was the genuine blocker; single-agent asks were polish.

## Longed for (open — tooling)
- **SDF `.label()` regions don't surface in `model:preview` JSON** (2/5 — kid,
  wizard). `labels: []` even when every region is labelled, so CLI agents can't
  verify paint coverage without launching the app. (In-app `paintByLabels`
  resolved every region correctly at bake time — confirmed — so this is a
  preview-tool gap, not an engine bug.) Worth surfacing SDF label names in
  `scripts/previewModel.ts` stats so the headless loop can self-check paint
  regions. Filing as a backlog item rather than expanding this PR's scope.
- **A few pose niceties** (single-agent, deferred): wrist/hand orientation roll
  (waving palm-forward), a tiptoe/ankle joint, and richer hair styles
  (`length`/shoulder-length) than the current `long` back-blob.
